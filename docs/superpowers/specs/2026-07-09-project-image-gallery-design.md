# Project Image Gallery Design

## Goal

프로젝트 카드에서 대표 화면을 빠르게 인지하고, 상세 팝업에서 실제 서비스 화면을 원본 비율에 가깝게 확인할 수 있도록 한다.

## Selected Direction

사용자가 선택한 B안인 `대표 이미지 카드 + 상세 팝업 갤러리`를 적용한다.

- 프로젝트 카드 상단에는 프로젝트별 대표 이미지 한 장을 표시한다.
- 카드 이미지는 동일한 높이를 유지하되 전체 화면이 과도하게 잘리지 않도록 `object-contain`을 사용한다.
- 상세 팝업 상단에는 선택된 이미지를 크게 표시한다.
- 이미지가 두 장 이상이면 하단 썸네일을 클릭해 큰 이미지를 전환한다.
- 이미지가 한 장이면 썸네일 영역을 표시하지 않는다.

## Data Model

`ProjectProps`에 선택적 `images: string[]` 필드를 추가한다. 첫 번째 이미지는 카드 대표 이미지이자 갤러리 초기 이미지다. 기존 `imgSrc`는 이전 데이터와의 호환을 위해 유지한다.

프로젝트 이미지 매핑:

- DocQ: `/images/project/DocQ.png`, `/images/project/DocQ2.png`
- 우주드로우: `/images/project/우주드로우.png`
- Y-ECOUNT: `/images/project/y-ecount.png`

## Components

### ProjectItem

`images[0]`을 우선 사용하고 없으면 `imgSrc`를 사용한다. 이미지 영역은 안정적인 고정 높이와 중립 배경을 사용한다.

### ProjectModal

프로젝트가 바뀔 때 첫 번째 이미지로 선택 상태를 초기화한다. 큰 이미지에는 프로젝트명과 이미지 순서를 포함한 대체 텍스트를 제공한다. 썸네일 버튼은 선택 상태가 시각적으로 구분되어야 한다.

## Responsive Behavior

- 데스크톱: 상세 팝업의 핵심 문장 아래에 넓은 갤러리를 배치한다.
- 모바일: 갤러리는 화면 너비를 사용하고 이미지 높이는 제한해 상세 내용 접근을 방해하지 않는다.
- 모든 이미지에 `object-contain`을 적용해 서로 다른 비율의 화면이 잘리지 않게 한다.

## Verification

- 세 프로젝트 카드에 올바른 대표 이미지가 표시되는지 확인한다.
- DocQ 상세 팝업에서 두 이미지 전환이 되는지 확인한다.
- 우주드로우와 Y-ECOUNT에서는 썸네일 영역이 생기지 않는지 확인한다.
- 다크 모드와 모바일 너비에서 이미지 경계 및 텍스트가 겹치지 않는지 확인한다.
- TypeScript 검사와 프로덕션 빌드를 실행한다.
