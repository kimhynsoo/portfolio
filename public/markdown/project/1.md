### 문제 정의

집, 나무, 사람 그림을 하나의 통합 모델로 탐지할 때 도메인별 입력 차이로 분석 편차가 발생할 수 있었습니다. AI 모델 결과가 서비스 리포트로 이어지는 구조였기 때문에 추론 흐름의 안정성이 중요했습니다.

### 해결

- YOLO 기반 집, 나무, 사람 객체탐지 모델을 도메인별로 분리했습니다.
- FastAPI 기반 AI 분석 서버와 추론 API를 구현했습니다.
- OpenCV로 이미지 특징 추출 파이프라인을 구성했습니다.
- S3 Presigned URL 기반 업로드, Docker Compose 실행 환경, Nginx Reverse Proxy와 SSL 배포 환경을 구성했습니다.

### 강조점

단순히 모델을 붙이는 데서 끝내지 않고, 입력 도메인별 추론 흐름을 분리해 AI 분석 결과가 서비스 안에서 더 안정적으로 사용되도록 설계했습니다.
