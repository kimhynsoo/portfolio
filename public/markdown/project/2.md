### 문제 정의

ERP 데이터를 자연어로 조회하는 기능에서는 LLM이 생성한 SQL을 그대로 실행하면 권한, 데이터 범위, 파괴적 쿼리 실행 위험이 생길 수 있습니다.

### 해결

- NLQ Assistant가 생성한 SQL을 직접 실행하지 않고 검증 단계를 거치도록 설계했습니다.
- AST 검증, 허용 테이블 화이트리스트, SELECT 실행 제한을 적용했습니다.
- 회사 ID는 클라이언트 입력이 아니라 서버에서 주입하도록 분리했습니다.
- Workflow Engine으로 요청 해석, 쿼리 생성, 검증, 실행 흐름을 단계별로 관리했습니다.

### 강조점

AI 기능을 구현하는 것보다, AI가 만든 결과를 서비스에서 안전하게 통제하는 구조를 설계하는 데 초점을 맞췄습니다.
