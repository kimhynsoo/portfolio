### 문제 정의

1차 MVP 배포 후 사용자마다 게임 진행 화면이 다르게 보이는 문제가 발생했습니다. WebSocket은 실시간 이벤트 전달에는 적합했지만, 각 사용자의 응답 처리 완료 여부를 보장하기에는 부족했습니다.

### 해결

- WebSocket은 실시간 연결을 담당하고, Redis Stream은 메시지 처리 신뢰성과 상태 일관성을 담당하도록 역할을 분리했습니다.
- Consumer Group과 ACK로 사용자 응답 처리 여부를 추적했습니다.
- Pending 메시지 재처리와 ACK Timeout 10초를 적용해 미응답 사용자를 자동 처리했습니다.
- 27개 사용자 피드백을 기능, UX, 안정성으로 분류하고 핵심 게임 흐름에 영향이 큰 개선부터 반영했습니다.

### 성과

- 사용자 간 화면 진행 상태 불일치 문제 해결
- 사용자 피드백 92% 반영
- 이후 테스트 피드백에서 화면 불일치 관련 불만 0건
- SSAFY 구미 2반 공통프로젝트 1위
